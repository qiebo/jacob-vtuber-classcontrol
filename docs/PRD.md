## 1. 背景与目标

### 1.1 背景

当前系统已具备"教师端管控 + 学生端 VTuber"基础能力，但存在以下痛点：

- 学生端缺少用户身份管理，不同学生共用同一台机器时无法隔离作品。
- 教师端缺少班级管理，无法按班级组织学生和批量操作。
- 教师端以系统浏览器打开管控界面，缺少桌面应用形态，且功能设置常驻侧栏，课堂使用体验欠佳。
- 作品收集无法展示进度，学生自定义形象与默认素材未做区分打包。

### 1.2 目标

围绕 **"班级教学、分组共用机器"** 这一核心场景，将系统升级为：

1. **学生端**：以用户名（设备名）登录，按用户隔离档案、文件、知识库，支持一键打包/恢复。
2. **教师端**：以桌面客户端形态运行，支持登录鉴权、班级管理、自动扫描、低频缩略图监控、进度化收集。
3. **平台兼容**：Windows 10 1903+（WebView2 预装）教师机；树莓派 OS（Python 3.10–3.12）学生机。

### 1.3 非目标（本期不做）

- 旧档案迁移（`profile_id = class_slug__student_slug` 结构的存量数据不迁移，全新开始）。
- Live2D 模型自定义打包判定（本期一律全量打包，后续版本再加"默认模型不可删"机制）。
- 跨网段/外网部署。
- 多教师协同、权限分级。

## 2. 用户角色与典型场景

### 2.1 角色定义

| 角色 | 终端 | 主要职责 |
|------|------|----------|
| 教师 | Windows 教师机 | 设备/班级/学生管理，课堂管控（锁定/分发/收集） |
| 学生 | 树莓派（小组共用） | 登录后自定义数智人形象/声音/知识库，开展活动并提交作品 |

### 2.2 典型场景

**场景 A：新学期首次使用**
1. 教师启动教师端 EXE → 登录（默认 `ybszr/123456`，可改）。
2. 系统自动扫描局域网在线设备。
3. 学生在树莓派上用设备名创建用户名（首次创建），登录进入默认配置。
4. 教师在"未分班"列表看到新学生，手动归入班级。

**场景 B：不同班级轮换使用同一台机器**
1. A 班学生甲登录 → 工作 → 退出（系统提醒保存）。
2. B 班学生乙在同一台机器上用另一用户名登录 → 载入乙的档案。
3. 前端不显示历史用户名，乙看不到甲的用户名痕迹。

**场景 C：课堂管控**
1. 教师在设备总览查看实时缩略图与状态。
2. 全选 → 锁定 → 分发任务文件 → 学生完成 → 解锁 → 一键收集作品（显示进度条）。
3. 作品自动按用户名命名，可再分发给其他学生。

**场景 D：学期末复位**
1. 教师使用"恢复初始设置"功能 → 清空所有学生文件，只保留默认文档。
2. 机器恢复到初始状态，供下一学期使用。

## 3. 功能需求

> 编号规则：`S-` 学生端、`T-` 教师端、`F-` 非功能。每条需求含验收标准。

### 3.1 学生端功能

#### S-1 用户名登录界面（P0）

**描述**：软件启动后先进入登录界面，而非直接进入主界面。

- 登录界面元素：用户名输入框、登录按钮、创建用户名按钮。
- 输入框不显示历史记录（`autocomplete="off"`），不保留下拉历史。
- 用户名规则：仅字母与数字，1–32 字符，前后去空格。

**验收**：启动应用首先看到登录界面；输入框无任何历史提示。

#### S-2 创建用户名（P0）

**描述**：首次使用时创建用户名。

- 提交时先校验本地注册表（离线去重），再请求教师机校验全局唯一性。
- 教师机在线且校验通过 → 创建成功。
- 教师机离线 → 走离线降级策略（见 S-3）。
- 创建后自动登录并使用默认配置（默认形象/背景/声音/性格）。

**验收**：
- 重名时提示"用户名已存在"并阻止创建。
- 教师机在线时，新用户名能被教师端看到。
- 创建后进入主界面，角色为默认配置。

#### S-3 离线降级创建（P0）

**描述**：教师机不可达时，允许本地创建带"临时标记"的用户名。

- 本地注册表去重通过后，创建用户并标记 `pending_sync=true`。
- 主界面顶部显示提示条："离线创建，待教师机同步"。
- 教师机恢复后，学生端在后台自动上报该用户名。
- **冲突处理**：若教师机已存在同名用户名（其他机器先创建），则提示学生改名；改名后本地档案目录重命名，数据保留。

**验收**：
- 断开教师机仍能创建并使用。
- 教师机恢复后，用户名出现在教师端列表。
- 模拟冲突时，学生端能正确提示并引导改名。

#### S-4 已有用户登录载入（P0）

**描述**：已有用户登录后，载入其最近一次保存的状态。

- "最近一次状态" = 上次退出时自动保存 + 期间任何手动保存。
- 退出时系统提醒保存（见 S-8）。
- 载入内容包括：角色配置（形象/背景/性格/声音）、知识库、聊天记录、个人文件。

**验收**：同一用户在两次会话间的自定义设置能被完整恢复。

#### S-5 退出登录（P0）

**描述**：学生可退出当前账号，返回登录界面。

- 退出前弹出确认框："是否保存当前作品？"（选项：保存并退出 / 不保存退出 / 取消）。
- 退出后清空当前会话状态，输入框不残留用户名。

**验收**：退出后回到登录界面，无残留数据。

#### S-6 个人文件绑定用户名（P0）

**描述**：每个用户的个人文件（知识库、聊天记录、上传作品）与用户名绑定，登录自动加载。

- 文件存储路径：`classroom_data/profiles/{username}/`
- 切换用户 = 切换整个工作区，互不干扰。

**验收**：A 用户上传的文件在 B 用户登录后不可见。

#### S-7 手动保存与加载状态（P1）

**描述**：用户可手动保存当前状态为"存档点"，并从存档点列表加载。

- 存档点列表仅显示当前用户名的存档（看不到他人存档）。
- 存档点存储在用户目录下的 `saves/` 子目录，按时间戳命名。

**验收**：
- 保存后列表新增一条；加载后状态正确恢复。
- 切换到另一用户，看不到前者的存档列表。

#### S-8 一键打包与恢复（P0）

**描述**：将当前完整状态（形象/背景/性格/声音/知识库/聊天记录/文件）打包为单一 ZIP。

- 本期统一全量打包（含素材）。
- 收到来自教师机的分发 ZIP 时，提示"检测到新作品，是否载入？"，确认后一键恢复。

**验收**：
- 打包后 ZIP 解压内容完整。
- 收到分发 ZIP 后能正确提示并恢复。

#### S-9 退出提醒保存（P0）

**描述**：用户退出登录或关闭应用时，提醒保存当前作品。

- 已在第 S-5 定义。教师机手动收集或学生手动提交时，也会保存一份。

**验收**：退出、收集、提交三种触发都产生有效保存。

### 3.2 教师端功能

#### T-1 桌面客户端形态（P0）

**描述**：EXE 启动后以无边框/无地址栏/无书签栏的桌面窗口形态呈现，模拟原生应用。

- 技术选型：**pywebview**（系统 WebView2）。
- 窗口标题："Jacob VTuber 课堂管控台"。
- 提供桌面快捷方式启动。

**验收**：启动后窗口无浏览器 chrome；可最小化/最大化/关闭。

#### T-2 教师登录鉴权（P0）

**描述**：打开界面首先看到登录页。

- 默认账号 `ybszr` / `123456`，密码可在设置中修改。
- 凭证校验通过后进入主界面，否则提示错误。
- 登录态保持：会话级（关闭窗口需重新登录）或持久化（勾选"记住我"），默认会话级。

**验收**：默认账号可登录；错误密码被拒；改密码后新密码生效。

#### T-3 自动扫描与设备列表（P0）

**描述**：登录后在后台自动扫描在线设备并按班级分组显示。

- 扫描频率：后台低频（默认 60 秒一次），用户可手动触发"立即扫描"。
- 设备卡片显示：在线状态、用户名、班级、缩略图、状态标签（保存/提交/锁定）、延迟、最后在线。
- 未分班的设备/学生单独分组显示。

**验收**：
- 设备上线/下线能在合理时间内反映。
- 卡片信息完整且实时（状态 5 秒级，缩略图 30 秒级）。

#### T-4 班级管理（P0）

**描述**：教师可创建、重命名、删除班级。

- 创建班级：输入名称（1–32 字符）。
- 学生归班：选中"未分班"学生 → 选择班级 → 确认。
- 移动学生：选中学生 → 移动到其他班级。
- 删除班级：班级下的学生回到"未分班"，不删除学生数据。

**验收**：
- 班级 CRUD 正常。
- 学生在班级间移动后，其用户名和档案不变（仅班级标签变化）。

#### T-5 低频缩略图监控（P0）

**描述**：以低资源占用刷新学生端屏幕缩略图。

- 缩略图轮询：**30 秒**一次，与状态文字轮询解耦。
- 状态文字（在线/锁定/保存）：5 秒一次。
- 缩略图采用懒加载 + 失败重试，单台失败不阻塞其他。

**验收**：
- 16 台并发缩略图更新不影响主界面流畅度。
- 缩略图时间戳正确反映最新快照。

#### T-6 管控操作保留（P0）

**描述**：保留现有管控功能并增强。

- 单台/批量：锁定、解锁、分发文件（≤50MB）。
- 一键收取学生作品（ZIP 包），**带进度条**。

**验收**：
- 锁定/解锁立即生效。
- 分发 50MB 文件到 16 台成功。
- 收取过程显示每台进度（0–100%）和总进度。

#### T-7 作品命名与分发（P0）

**描述**：收取的作品 ZIP 自动按学生用户名命名。

- 文件名：`{username}.zip`，含日期子目录 `%LOCALAPPDATA%\JacobTeacherConsole\collections\YYYY-MM-DD\`。
- 收取的作品可再分发给其他学生，接收方提示载入。

**验收**：
- 收取后文件名正确。
- 分发到学生端后，学生端弹出载入提示。

#### T-8 恢复初始设置（P1）

**描述**：教师端提供"恢复初始设置"功能。

- 操作对象：选中的单台或多台学生机。
- 效果：清空该机器上所有学生文件（profiles 目录），只保留默认文档。
- 二次确认："此操作将删除 {N} 个学生档案，不可恢复，确认？"

**验收**：
- 执行后学生机 `profiles/` 目录只剩默认文档。
- 学生端重启后回到登录界面（无用户）。

#### T-9 设置可收起（P0）

**描述**：将常驻左侧的设置功能收起，或移到独立选项卡。

- 默认隐藏设置，点击"设置"按钮才展开（抽屉式或独立页）。
- 设置内容：密码修改、扫描频率、缩略图频率、默认账号配置。

**验收**：设置不常驻占位；点击可展开；修改即时生效。

## 4. 业务流程

### 4.1 学生端登录与创建流程

```
启动应用
  │
  ▼
[登录界面] ──创建用户名──▶ [创建界面]
  │                           │
  │                      本地注册表去重
  │                           │
  │                      请求教师机校验
  │                           │
  │                     ┌─在线─▶ 全局去重通过 ▶ 创建成功 ▶ 自动登录
  │                     │
  │                     └─离线─▶ 本地去重通过 ▶ 创建(pending_sync) ▶ 自动登录
  │
  └─登录──▶ 本地注册表查找
                │
           ┌─存在─▶ 载入最近状态 ▶ 主界面
           │
           └─不存在─▶ 提示"用户名不存在，是否创建？"
```

### 4.2 教师端核心流程

```
启动 EXE
  │
  ▼
[登录页] ──ybszr/123456──▶ [主界面]
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
        [设备总览]        [班级管理]        [设置]
              │
    ┌─────────┼─────────┬─────────┐
    ▼         ▼         ▼         ▼
  扫描     批量锁定   分发文件   收取作品(进度)
  (60s)    /解锁      (≤50MB)     │
                                  ▼
                          collections/YYYY-MM-DD/{username}.zip
                                  │
                                  ▼
                          可再分发给学生
```

### 4.3 用户名唯一性仲裁流程

```
学生端发起创建 (username)
  │
  ▼
本地注册表查询 ──命中──▶ 拒绝（"用户名已存在"）
  │
未命中
  ▼
请求教师机 POST /classroom/users/check ──失败(网络)──▶ 离线降级创建(pending_sync)
  │
教师机在线
  ▼
教师机查询 users.json ──命中──▶ 拒绝
  │
未命中
  ▉
教师机登记 + 返回成功
  │
学生端创建本地档案 + 自动登录
```

### 4.4 离线创建同步与冲突处理流程

```
学生端 pending_sync 用户 ──教师机恢复在线──▶ 后台自动上报
                                              │
                                     ┌─教师机无冲突─▶ 标记 synced
                                     │
                                     └─教师机已存在同名─▶ 提示学生改名
                                                              │
                                                   改名成功 ▶ 本地目录重命名 + 数据保留
```

## 6. 数据概念模型（预览）

> 详细字段见开发文档，此处仅列概念。

```
User（用户名注册表，教师机）
  ├─ username (PK, 仅字母数字, 全局唯一)
  ├─ class_id (可空)
  ├─ device_hint (首次创建所在设备，可空)
  ├─ created_at
  └─ status (active / pending_sync)

Class（班级，教师机）
  ├─ class_id (PK)
  └─ name

Device（设备，教师机，沿用现有）
  ├─ device_id (PK)
  ├─ name, base_url, group, enabled, token, ...

Profile（学生档案，树莓派）
  ├─ username (PK)               ← 取代原 profile_id
  ├─ class_name (仅显示用)
  ├─ character_config
  ├─ workspace_state
  ├─ created_at, updated_at, last_saved_at
  ├─ dirty, submitted
  └─ pending_sync (离线创建标记)
```

## 7. 验收标准（系统级）

1. **学生端**：新用户创建 → 默认配置 → 自定义 → 退出保存 → 重新登录载入，全链路数据一致。
2. **教师端**：登录 → 自动扫描 → 班级管理 → 批量管控 → 进度化收取，全链路无阻塞。
3. **共用机器**：用户 A 退出 → 用户 B 登录，互不可见对方数据。
4. **离线**：断开教师机，学生仍能创建并工作；教师机恢复后自动同步。
5. **复位**：恢复初始设置后，机器回到出厂状态。
6. **并发**：16 台学生机同时操作，教师端无卡顿、无丢任务。

## 8. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|:--:|:--:|------|
| WebView2 运行时缺失 | 低 | 高 | 首次启动检测并提示安装；安装包随 EXE 分发 |
| 教师机单点故障导致全班停摆 | 中 | 高 | 离线降级策略（S-3）；关键操作本地优先 |
| 同名用户并发创建冲突 | 中 | 中 | 教师机作为唯一仲裁源；离线创建标记 pending_sync |
| 单机多用户磁盘膨胀 | 中 | 中 | 恢复初始设置功能；后续版本加存档数量上限 |
| 学生误删自己的存档 | 中 | 低 | 删除二次确认；存档不在默认文档范围 |

## 附录 A：术语表

| 术语 | 含义 |
|------|------|
| 用户名 | 学生登录标识，规则：仅字母数字，1–32 字符，对应设备名 |

| 作品 | 学生完整状态的 ZIP 包，用于收取/分发/恢复 |
| 恢复初始设置 | 清空所有学生文件，只保留默认文档 |
