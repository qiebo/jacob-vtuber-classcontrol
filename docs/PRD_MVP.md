# Jacob VTuber 课堂管控系统 — 最小可交付版 PRD（MVP）

| 项目 | 内容 |
|------|------|
| 文档版本 | MVP v1.0 |
| 编写日期 | 2026-06-18 |
| 状态 | 已按联调结果调整 |
| 适用范围 | 树莓派学生端 + Windows 教师端 |
| 设计原则 | 先稳定可用，暂缓班级/复杂管理功能 |

---

## 1. 背景与目标

原升级版规划覆盖了用户名、班级、扫描、缩略图、作品收集、存档点、离线同步、教师端登录等较完整功能。但经过联调与课堂场景重新评估后，当前阶段应优先交付一个**更稳定、更少干扰、更容易操作的课堂最小可用版本**。

本 MVP 的目标不是做完整教务管理系统，而是完成课堂教学的核心闭环：

> 学生能用用户名登录并恢复自己的作品；教师能发现学生机、锁屏/解锁、分发文件、收取作品、下发作品包。

---

## 2. MVP 核心范围

### 2.1 教师端保留功能

教师端只保留课堂中最常用、最必要的能力：

1. 自动扫描学生机
2. 锁屏 / 解锁控制
3. 文件分发
4. 学生作品收取
5. 下发单独学生作品包，一键应用到选中的学生机上
6. 显示在线状态、当前用户名、错误信息、收取结果

### 2.2 学生端保留功能

学生端只保留以用户名为核心的个人工作区能力：

1. 用户名创建与登录
2. 同一个用户名的状态保存与恢复
3. 当前学生作品打包上传到教师端
4. 接收教师下发作品包，并确认是否载入
5. 退出登录前保存提示

### 2.3 暂缓功能

以下功能暂缓，不作为 MVP 验收条件：

| 功能 | 原因 |
|------|------|
| 班级管理 / 分班 / 归班 | 当前设计尚未充分确认，避免干扰最小可用流程 |
| 复杂缩略图墙 | 可保留底层缩略图能力，但不作为核心依赖 |
| 全局用户名强仲裁 UI | 后续再结合教师端用户管理重新设计 |
| 离线冲突改名 UI | 后续再做 |
| 多教师协同 | 不做 |
| 复杂设置页 / 设置抽屉 | 后续优化 |
| 自动分组 / 自动归班 | 暂缓 |
| 班级维度统计与管理 | 暂缓 |

## 3. 典型课堂流程

### 3.1 上课前

1. 教师启动教师端。
2. 教师端自动扫描局域网学生机。
3. 学生启动树莓派端。
4. 学生输入用户名登录；首次使用时创建用户名。
5. 教师端看到在线设备与当前用户名。

### 3.2 课堂中

1. 教师可批量锁屏，讲解任务。
2. 教师分发任务文件或素材。
3. 教师解锁，学生制作数智人作品。
4. 学生可手动保存当前状态。
5. 教师可再次锁屏、解锁或检查设备状态。

### 3.3 下课前

1. 教师发起作品收取。
2. 学生端先自动保存当前状态。
3. 学生端打包当前用户名工作区为 ZIP。
4. 教师端下载作品包，并显示每台设备收取结果。

### 3.4 下次课

1. 学生用同一用户名登录。
2. 系统恢复上次保存状态。
3. 学生继续制作。

### 3.5 作品示范 / 互评

1. 教师选中一个学生作品 ZIP。
2. 教师选择若干学生机。
3. 教师下发该作品包。
4. 学生端弹出确认：是否载入教师下发作品。
5. 学生确认后，当前工作区恢复为该作品包内容。

## 4. 功能需求

### S-1 学生端用户名登录（P0）

**描述**：学生端启动后必须先显示登录页，不能自动登录上一次用户。

**要求**：
- 输入框：用户名。
- 按钮：登录、创建用户名。
- 输入框不显示历史用户名，不自动补全。
- 用户名规则：仅字母和数字，1–32 字符。
- 后端启动时清空 `current_username` 和 `session_token`。

**验收**：
- 重启树莓派端后看到登录页。
- 输入框为空。
- 不会自动进入上一个用户。

---

### S-2 创建用户名（P0）

**描述**：首次使用时创建用户名。

**要求**：
- 本地检查是否已存在。
- 不依赖教师端即可创建。
- 创建后进入主界面。
- 目录结构：`classroom_data/profiles/{username}/`。

**验收**：
- 合法用户名可创建。
- 非法用户名被拒。
- 创建后能进入主界面。
- profile.yaml 中含 `username` 字段。

---

### S-3 登录已有用户名并恢复状态（P0）

**描述**：同一个用户名再次登录时恢复上次保存状态。

**要求**：
- 登录后应用对应 `profile.character_config`。
- 登录后应用 `workspace_state`。
- 人物形象、背景、文件、知识库等应恢复。

**验收**：
- 学生 A 修改形象后保存。
- 退出并重新登录 A。
- 人物形象恢复正常。

---

### S-4 保存当前作品（P0）

**描述**：学生可保存当前状态。

**要求**：
- 保存 character_config。
- 保存 workspace_state。
- 保存知识库、聊天记录、个人文件引用。
- 更新 `last_saved_at`。

**验收**：
- 保存后刷新或重新登录，状态不丢失。

---

### S-5 退出登录前保存提示（P0）

**描述**：学生退出登录前需确认是否保存。

**要求**：
- 弹出提示：是否保存当前作品。
- 选项：保存并退出、不保存退出、取消。
- 退出后清空当前登录态，返回登录页。

**验收**：
- 保存并退出后，重新登录能恢复。
- 不保存退出后，不写入新状态。
- 取消后仍停留当前页面。

---

### S-6 当前作品打包（P0）

**描述**：学生端能将当前用户名工作区打包为 ZIP。

**要求**：
- ZIP 包含 profile.yaml、manifest.json、assets、chat_history、files、snapshot。
- 文件名 `{username}.zip`。
- 打包前可自动保存当前状态。

**验收**：
- 教师端收取 ZIP。
- ZIP 内 profile.yaml 含 `username`。
- 解包内容完整。

---

### S-7 接收教师下发作品包（P0）

**描述**：学生端收到教师下发的作品 ZIP 后提示是否载入。

**要求**：
- 不应静默覆盖当前工作区。
- 弹窗确认：是否载入教师下发作品。
- 确认后恢复 ZIP 内容。
- 恢复前如有未保存修改，应提示保存。

**验收**：
- 教师端下发作品包。
- 学生端出现确认提示。
- 确认后作品状态恢复。

---

### T-1 自动扫描学生机（P0）

**描述**：教师端自动发现/刷新学生机状态。

**要求**：
- 支持手动刷新。
- 支持后台定时刷新。
- 显示设备在线/离线、当前用户名、最后在线、错误原因。
- 不强依赖班级。

**验收**：
- 树莓派上线后教师端能看到。
- 树莓派登录用户名后教师端能显示当前用户名。
- 设备离线后显示离线或错误。

---

### T-2 锁屏 / 解锁（P0）

**描述**：教师可对单台或多台学生机锁屏/解锁。

**要求**：
- 单台锁屏/解锁。
- 批量锁屏/解锁。
- 单台失败不影响其他设备。

**验收**：
- 锁屏后学生端不能操作。
- 解锁后恢复操作。
- 批量操作返回每台结果。

---

### T-3 文件分发（P0）

**描述**：教师向选中学生机分发普通文件。

**要求**：
- 支持选择多个设备。
- 文件大小限制建议 50MB。
- 上传到当前用户个人文件区。

**验收**：
- 选中设备后分发文件。
- 学生端对应用户名文件区可看到文件。

---

### T-4 学生作品收取（P0）

**描述**：教师从选中学生机收取当前登录用户名的作品 ZIP。

**要求**：
- 收取前学生端自动保存。
- 按当前用户名导出 ZIP。
- 教师端保存到日期目录。
- 收取结果显示成功/失败。

**验收**：
- 收取文件名包含 username。
- ZIP 内容完整。
- 离线设备显示失败，不阻塞其他设备。

---

### T-5 下发作品包并应用（P0）

**描述**：教师将一个学生作品 ZIP 下发给选中学生机，学生确认后载入。

**要求**：
- 教师端选择 ZIP。
- 教师端选择目标设备。
- 学生端收到后提示确认。
- 确认后调用 workspace restore。

**验收**：
- 教师下发作品包。
- 学生端弹出确认。
- 确认后人物/背景/文件状态恢复。

---

### T-6 教师登录（P1）

**描述**：教师端默认账号登录即可。

**要求**：
- 默认账号：`ybszr`。
- 默认密码：`123456`。
- 支持改密码。

**验收**：
- 正确密码登录。
- 错误密码拒绝。

## 5. 非功能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| F-1 | 稳定性优先 | 核心流程不能依赖班级、缩略图等非必要模块 |
| F-2 | 局域网可用 | 教师机和学生机在同一局域网内通信 |
| F-3 | 单点失败容错 | 单台失败不影响批量操作 |
| F-4 | 数据隔离 | 不同 username 的工作区互不干扰 |
| F-5 | 可恢复 | 树莓派重启后必须回登录页，不自动登录 |
| F-6 | 可备份 | 学生作品以 ZIP 形式收取 |

---

## 6. MVP 完成定义

MVP 完成必须满足：

- [ ] 树莓派端启动后显示登录页，不自动登录。
- [ ] 用户名创建与登录正常。
- [ ] 同用户名保存/恢复正常。
- [ ] 教师端能自动扫描学生机并显示当前用户名。
- [ ] 教师端能锁屏/解锁。
- [ ] 教师端能分发普通文件。
- [ ] 教师端能收取当前学生作品 ZIP。
- [ ] 教师端能下发学生作品 ZIP，学生端确认后应用。
- [ ] 收取结果能显示成功/失败。
- [ ] 树莓派端语音识别正常。
- [ ] 无班级功能入口干扰主流程。

---

## 7. 当前已知取舍

1. **班级管理暂缓**：后端可保留，但前端不展示，避免误导。
2. **缩略图不是核心验收项**：可用于辅助观察，但不作为课堂闭环依赖。
3. **VAD 服务端关闭**：避免 torch 依赖，沿用旧版行为，由前端/ASR 流程处理。
4. **ASR 走 CPU**：树莓派 5 GPU 不支持通用计算，无法用 GPU 加速本地 ASR。
5. **VNC 横屏优先**：使用 VNC 时不看物理屏，DSI-2 不旋转，VNC 显示横屏。
