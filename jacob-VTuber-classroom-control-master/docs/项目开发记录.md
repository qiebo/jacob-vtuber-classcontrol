# 项目开发记录

更新时间：2026-04-03

## 1. 项目定位

本项目基于 `Open-LLM-VTuber` 做了教学场景和 kiosk 场景定制，目标不是做一个通用开发框架，而是做一个可在树莓派和 Ubuntu 小主机上稳定运行、可直接交给教师或现场人员使用的数智人应用。

当前版本已经同时支持两种舞台角色模式：

1. `Live2D` 模式
2. `图片/视频（AvatarPack）` 模式

项目重点不是单纯“能跑”，而是：

1. 前端可视化配置
2. 语音对话链路稳定
3. 角色素材和舞台素材可管理
4. 局域网可访问，便于调试和验收
5. AvatarPack 素材规则清晰、统一、可交接

## 2. 当前已实现的核心功能

### 2.1 舞台与角色

1. 保留原有 `Live2D` 数智人模式
2. 新增 `图片/视频` 模式
3. 两种模式可切换
4. `图片/视频` 模式支持角色包预览、启用、删除
5. `Live2D` 模式支持模型列表、切换、自定义模型上传
6. 设置页中已加入 `图片/视频` 与 `Live2D` 模式切换入口

### 2.2 AvatarPack 上传与运行

1. 支持上传 `png/jpg/jpeg/webp/gif/zip`
2. 不再支持 `mp4/webm` 直传
3. 支持将整理好的整套素材打成一个 `zip` 上传
4. 上传后可一键处理为统一运行格式 `AvatarPack`
5. 自动生成：
   - `manifest.json`
   - `actions/<state>/*.png`
   - `actions/action_01/*.png`
   - `actions/action_02/*.png`
   - `thumb.png`
6. 运行时按状态机驱动动作切换：
   - `idle`
   - `listening`
   - `thinking`
   - `speaking`
   - `action`
7. `idle` 为必选状态，其它状态缺失时统一回退到 `idle`
8. `action` 缺失时点击不执行任何动作
9. `action` 支持多组，空闲状态下点击随机播放一组
10. 支持鼠标滚轮缩放、拖拽移动、点击互动

### 2.3 对话链路

1. 保留原有 ASR / LLM / TTS 链路
2. 已修正 `idle -> listening -> thinking -> speaking -> idle` 状态机
3. 回答结束后可按设置恢复麦克风
4. `语音交互` 页中的自动停麦/自动开麦开关已恢复有效
5. Live2D 与 AvatarPack 模式都能跟随对话状态切换

### 2.4 字幕与界面

1. 底部字幕已升级为“实时对话浮层”
2. 左右两侧分别展示用户与数智人消息
3. 字幕会滚动保留最近对话
4. 设置栏与历史栏已重新布局为左右双侧栏
5. 输入区、字幕区、悬浮按钮已做居中和悬浮样式优化
6. 左上角连接状态已改成“状态点 + 当前对话状态”

### 2.5 部署与运行

1. 树莓派已支持运行
2. Ubuntu 小主机也可按同一代码结构部署
3. 服务已支持监听 `0.0.0.0`，局域网浏览器可直接访问
4. 当前默认访问端口为 `12393`

  WebSocket 主消息分发，负责前后端实时通信。

## 4. 当前关键架构说明

### 4.1 角色模式架构

前端统一维护一个模式变量：

- `avatar_mode = live2d | avatarpack`

当模式切换时：

1. `live2d` -> 使用 Live2D 画布和模型链路
2. `avatarpack` -> 使用帧序列播放链路

这样做的好处是：

1. 保留原有 Live2D 功能
2. 新增模式时对旧链路侵入最小
3. 两种模式可以共享语音状态机和舞台 UI

### 4.2 AvatarPack 运行格式

当前统一运行格式如下：

```text
avatar_pack/{pack_id}/
  manifest.json
  thumb.png
  actions/
    idle/
    listening/
    thinking/
    speaking/
    action_01/
    action_02/
```

`manifest.json` 当前至少包含：

1. `pack_id`
2. `name`
3. `version`
4. `canvas.width / canvas.height`
5. `anchor`
6. `actions`
7. `action_groups`
8. `fallback_map`
9. `thumb`

### 4.3 状态机架构

当前前端运行时状态链路为：

```text
idle -> listening -> thinking -> speaking -> idle
```

动作映射原则：

1. 空闲时优先 `idle`
2. 倾听时优先 `listening`，缺失回退 `idle`
3. 思考时优先 `thinking`，缺失回退 `idle`
4. 说话时优先 `speaking`，缺失回退 `idle`
5. 点击时仅在 `idle` 状态下触发 `action`
6. `action` 有多个时随机播放一个
7. `action` 缺失时点击无效果

### 4.4 前后端通信架构

#### REST

REST 负责资源管理类操作，例如：

1. 上传 AvatarPack 素材
2. 上传 Live2D 模型
3. 获取角色目录
4. 上传背景图

#### WebSocket

WebSocket 负责实时对话和状态同步，例如：

1. 初始配置下发
2. 语音链路状态变更
3. 对话消息流
4. 麦克风启停控制

## 5. 当前项目重点与交接关注点

### 5.1 当前项目重点

1. **以运行稳定为第一优先级**
   已围绕树莓派课堂运行做过多轮修复，重点是状态机、麦克风、TTS、界面切换稳定。

2. **AvatarPack 是本项目新增重点**
   这是当前版本区别于原版项目的最重要扩展。

3. **局域网访问能力已打通**
   便于电脑、平板、手机直接查看页面效果。

4. **素材规则已收敛为唯一标准**
   对外只推荐 `idle/listening/thinking/speaking/action` 五类命名。

### 5.2 交接给他人时需要重点说明

1. 前端源码和打包产物都存在
   实际运行优先读取 `frontend-source/dist/web`。

2. 改前端后必须重新构建
   否则设备上看到的还是旧界面。

3. `图片/视频` 模式的素材规则已改为“严格命名驱动”
   当前不是智能识别动作，而是按命名规则分配动作。

4. AvatarPack 当前不做自动抠图
   上传有背景的素材，背景会一起保留。

5. AvatarPack 已取消视频直传
   教学和交接场景下统一由素材提供者先离线转成图片序列或 GIF，再整体打包上传。

## 6. 当前已知限制

1. AvatarPack 上传不做自动去背景
2. AvatarPack 不再负责视频抽帧
3. AvatarPack 默认布局仍然主要依赖通用规则，不是每个角色包都有独立人工校准
4. Live2D 预览图未做稳定单图方案，因此当前已去掉
5. 图片模式和 Live2D 模式的“手动调整后自动固化”为默认值逻辑，已做本地持久化，但交接时仍应实际回归验证

## 7. 推荐运维与开发流程

### 7.1 开发修改后

```bash
cd frontend-source
npm install
npm run build:web
cd ..
python run_server.py
```

### 7.2 树莓派部署后检查

1. 页面是否能打开
2. Live2D / AvatarPack 是否都能正常切换
3. 麦克风、打断、TTS 是否正常
4. AvatarPack zip 上传后是否能立即出现在角色列表
5. `action` 在空闲状态点击时是否能随机触发
