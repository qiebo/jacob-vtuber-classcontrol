# 图片/视频素材上传使用说明

更新时间：2026-04-03

本说明针对当前项目中“图片/视频（AvatarPack）模式”的**实际可用版本**编写，适用于：

1. 最终用户上传人物素材
2. 运维人员指导素材准备
3. 后续接手开发的同事理解当前规则

## 1. 功能概述

当前项目已实现：

1. 用户上传图片、GIF 或 zip 素材包
2. 系统自动处理成统一运行格式 `AvatarPack`
3. 上传成功后直接加入“图片/视频”角色列表
4. 可立即启用并在舞台上运行

这意味着：**现在已经实现了“素材上传以后，一键转换成可用人物形象”。**

## 2. 当前支持的素材格式

系统当前允许上传以下格式：

1. `png`
2. `jpg`
3. `jpeg`
4. `webp`
5. `gif`
6. `zip`

说明：

1. `png/jpg/jpeg/webp`
   视为单张图片，通常会变成某个状态的一帧。

2. `gif`
   会自动逐帧提取，作为一个状态的帧序列。

3. `zip`
   推荐使用。将按命名规则整理好的整套素材打包后一次上传，系统会自动解包并处理。

## 3. 当前不再支持的格式

以下格式当前**不再支持直传**：

1. `mp4`
2. `webm`

原因：

1. 教学场景下学生上传视频命名与切段容易混乱
2. 后端不再承担视频抽帧，规则更稳定
3. 统一改为先离线处理，再按命名规则打包上传

## 4. 上传大小限制

当前后端限制如下：

1. 单个文件最大：`200MB`
2. 单次上传总大小最大：`500MB`
3. 单次最多文件数：`300`
4. 单个 zip 解压后的总内容最大：`500MB`

超过限制会上传失败。

## 5. 上传入口使用方式

在设置页进入：

1. `舞台`
2. 选择 `图片/视频`
3. 点击 `上传人物素材`

然后按顺序操作：

1. 输入人物形象名称（自定义名字）
2. 拖拽素材到上传区，或点击 `+` 选择文件
3. 点击 `一键处理并启用`

上传完成后，新的图片角色会自动出现在图片/视频角色列表中。

## 6. “自定义名字”的作用

上传弹窗中的“自定义名字”就是当前 AvatarPack 的角色名。

当前行为可以理解为：

1. 同一次上传的所有素材
2. 在同一个“自定义名字”下
3. 会被归并为同一个人物形象包

例如：

- 角色名：`数学老师小翼`
- 上传文件：`idle_001.png`、`speaking_001.png`、`thinking_001.gif`

最终会生成一个名为“数学老师小翼”的图片角色包。

## 7. 当前最关键规则：文件名决定状态归属

当前版本不是通过 AI 自动理解素材属于什么状态，而是通过**文件名前缀**来判断素材对应哪个状态。

如果命名不对，上传会直接失败，不再默认归类到某个状态。

## 8. 标准状态模型

当前正式对外规则只保留 5 类状态：

1. `idle`
2. `listening`
3. `thinking`
4. `speaking`
5. `action`

其中：

1. `idle`
   - 必须有
   - 表示空闲待机状态

2. `listening`
   - 可选
   - 表示用户说话中、系统收音中

3. `thinking`
   - 可选
   - 表示识别完成后、播报开始前的思考状态

4. `speaking`
   - 可选
   - 表示 TTS 播放中的说话状态

5. `action`
   - 可选
   - 表示鼠标点击互动动作
   - 可有多个

## 9. 回退规则

### 9.1 必选规则

- `idle` 必须存在
- 如果没有 `idle`，上传失败，不生成角色包

### 9.2 状态回退

- `listening` 缺失 -> 回退到 `idle`
- `thinking` 缺失 -> 回退到 `idle`
- `speaking` 缺失 -> 回退到 `idle`

### 9.3 action 规则

- `action` 缺失 -> 点击不执行任何动作
- 不报错，不切换状态
- `action` 只在 `idle` 状态下触发
- `thinking / speaking / listening` 状态下点击无效

## 10. 单独上传时的命名规则

### 10.1 普通状态命名

推荐统一使用：

```text
idle_001.png
idle_002.png
listening_001.png
thinking_001.png
speaking_001.png
```

规则：

1. 前缀必须是状态名
2. 后缀数字表示帧序号
3. 同一状态下可上传 1 张或多张
4. 也支持单文件写法，例如 `idle.png`、`speaking.gif`

### 10.2 action 多动作命名

推荐统一使用三段式：

```text
action_01_001.png
action_01_002.png
action_01_003.png

action_02_001.png
action_02_002.png
```

含义：

1. `action_01` 表示第 1 个点击动作
2. `action_02` 表示第 2 个点击动作
3. 最后的 `001/002/003` 表示该动作内部的帧序号

如果一个动作只有一个 GIF，也可以：

```text
action_01.gif
action_02.gif
```

## 11. zip 上传规则

这是当前最推荐的上传方式。

### 11.1 zip 内部规则

zip 内部素材与单独上传时使用**完全相同**的命名规则。

推荐直接放文件，例如：

```text
idle_001.png
idle_002.png
listening_001.png
thinking_001.png
speaking_001.png
action_01_001.png
action_01_002.png
action_02_001.png
```

也允许放在子目录里，例如：

```text
my_pack/
  idle_001.png
  idle_002.png
  speaking_001.png
  action_01_001.png
```

程序会：

1. 解压 zip
2. 递归扫描其中的素材文件
3. 忽略目录名
4. 只按**文件名**判断状态归属

所以：

- 文件夹名不参与识别
- 是否放在子目录里都可以
- 最关键的是文件名要符合规则

### 11.2 zip 内部允许的素材格式

zip 内允许：

1. `png`
2. `jpg`
3. `jpeg`
4. `webp`
5. `gif`

zip 内不允许：

1. `mp4`
2. `webm`
3. 其它无关素材文件

如果 zip 内包含不支持的素材文件，上传会失败并提示错误。

## 12. 当前系统会自动做什么

上传后，系统会自动完成以下处理：

1. 校验文件类型
2. 校验文件大小
3. 校验素材命名规则
4. zip 自动解包
5. GIF 自动逐帧提取
6. 按文件名归类到不同状态
7. 对 `action_01 / action_02` 这类多动作自动分组
8. 统一整理为动作目录
9. 统一生成 `manifest.json`
10. 自动生成缩略图 `thumb.png`

## 13. 当前系统不会自动做什么

以下能力**当前版本没有实现**，上传前需要人工处理：

1. 自动去背景 / 自动抠图
2. 自动把视频切成多个状态
3. 自动识别人物主体并重新校准位置
4. 自动修复不同动作尺寸不一致的问题

## 14. 关于透明背景

这是非常重要的一点。

当前版本不会自动去背景，所以：

1. 如果上传的 `png/webp` 本身就是透明背景，效果最好
2. 如果上传的是有背景的图片或 GIF，背景会被一起保留

如果希望人物像“真正浮在舞台上”，建议：

1. 优先使用透明背景 `png`
2. 保证人物四周留白一致

## 15. 关于尺寸与人物位置

当前系统会把所有帧统一到一个画布尺寸中，但素材本身的位置一致性仍然很重要。

建议：

1. 同一个人物的所有状态尽量使用同样的宽高
2. 人物在画面中的位置尽量一致
3. 脚底、头顶、身体中心不要在不同状态间差太多

否则运行时切换状态时，人物可能会出现跳动。

## 16. action 的运行逻辑

当前 action 行为固定如下：

1. `action` 只在 `idle` 状态下触发
2. `thinking / speaking / listening` 状态下点击无效
3. 如果只有一个 `action`，点击直接播放这个动作
4. 如果有多个 `action`，点击时随机播放一个
5. `action` 播放结束后回到 `idle`

## 17. 运行后的目录结构

上传成功后，系统会自动生成人物包，大致结构如下：

```text
avatar_pack/{pack_id}/
  manifest.json
  thumb.png
  actions/
    idle/
      0001.png
      0002.png
    listening/
    thinking/
    speaking/
    action_01/
    action_02/
```

这就是当前项目内部真正使用的统一运行格式。

## 18. 给用户的最简操作建议

如果你不是开发人员，只想快速做一个可用人物，请按下面最简单方式准备：

1. 准备透明背景人物图
2. 至少命名成：

```text
idle_001.png
speaking_001.png
```

3. 如果还有思考图、倾听图和点击动作，再额外加：

```text
listening_001.png
thinking_001.png
action_01_001.png
action_01_002.png
```

4. 建议把整套素材打成一个 zip
5. 打开设置页 -> 舞台 -> 图片/视频 -> 上传人物素材
6. 填写人物名
7. 上传 zip 或这些文件
8. 点击 `一键处理并启用`

## 19. 给接手人员的结论

当前 AvatarPack 功能已经不是“占位方案”，而是已经落地可用：

1. 前端已有上传入口
2. 后端已有处理流水线
3. 运行时已有动作状态机
4. 上传后能直接生成并启用角色

同时，这一版规则已经做了收敛：

1. 正式状态名只保留 `idle/listening/thinking/speaking/action`
2. 对外推荐 zip 上传
3. 不再支持视频直传
4. `action` 支持多组随机播放

交接给其他人员时，**一定要先讲清楚命名规则和 zip 使用方式**。
