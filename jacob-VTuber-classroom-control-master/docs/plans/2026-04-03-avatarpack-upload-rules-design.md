# AvatarPack 上传规则收敛设计

日期：2026-04-03

## 目标

1. 对外只保留 `idle/listening/thinking/speaking/action` 五类状态
2. `idle` 必须有，其它状态缺失统一回退到 `idle`
3. `action` 缺失时点击无效果
4. `action` 允许多组，点击时随机播放一个
5. 取消 `mp4/webm` 直传，新增 `zip` 上传
6. zip 内外统一只按文件名识别状态，不依赖目录名

## 实施范围

1. 后端上传解析与 manifest 结构
2. AvatarPack 运行时动作选择逻辑
3. 前端上传弹窗文案与格式限制
4. 项目文档与交接说明

## 兼容策略

1. 旧包运行时继续兼容 `waiting/talk`
2. 新上传规则只对外展示新命名
3. 前端仍可播放旧 manifest 中的 `talk` 或旧单一 `action`

## 风险点

1. 旧包和新包共存时，运行时必须同时兼容两套字段
2. zip 需校验路径与解压大小，避免非法路径和超大内容
3. `action` 单帧资源也要能正常播放并自动回到 `idle`

## 验证重点

1. zip 上传是否成功解包并生成角色包
2. 缺失 `idle` 是否被拒绝
3. `speaking` 缺失时是否回退 `idle`
4. 多个 `action` 是否只在空闲状态触发且随机播放
5. 旧角色包是否仍可正常运行
